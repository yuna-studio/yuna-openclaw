## 피쳐 (1): 로그인 (Auth)
- **배경:** 유저 식별 및 웰컴 티켓 지급을 위한 관문.
- **목표:** 이탈 없는 0.1초 로그인 경험 제공.
- **가설:** 소셜 로그인(Google/Apple)만 제공함으로써 허들을 낮추면 전환율이 30% 상승할 것이다.
- **핵심 유저 스토리:** 유저는 복잡한 가입 없이 버튼 클릭 한 번으로 가입하고 웰컴 티켓을 확인한다.
- **디자인 방향성:** 럭셔리한 카드로 구성, 브랜드 로고 강조.
- **기술적 구현 방안:** Supabase Auth (OAuth 2.0) 사용.
- **데이터 로깅:** `login_click`, `login_success`, `login_fail`.
- **리스크:** OAuth 프로바이더 승인 지연.

## 피쳐 (2): 업로드 (Upload)
- **배경:** AI 생성을 위한 초음파 데이터 수집.
- **목표:** 정확한 얼굴 인식을 위한 고품질 사진 확보.
- **가설:** 가이드라인(정면 사진)을 명확히 제시하면 AI 생성 실패율이 50% 감소할 것이다.
- **핵심 유저 스토리:** 유저는 갤러리에서 초음파 사진을 선택하고 가이드에 맞춰 업로드한다.
- **디자인 방향성:** 직관적인 가이드 영역, 감성적인 프로그레스 바.
- **기술적 구현 방안:** Supabase Storage, Client-side Image Compression.
- **데이터 로깅:** `upload_start`, `upload_complete`, `upload_error`.
- **리스크:** 개인정보(태아 정보) 유출 위험.

## 피쳐 (3): 결제 (Payment)
- **배경:** 서비스 수익화를 위한 핵심 루프.
- **목표:** 5,900원의 가치를 인정받는 간편한 결제.
- **가설:** 무료 티켓 경험 후 결제 유도 시 구매 전환율이 높을 것이다.
- **핵심 유저 스토리:** 유저는 티켓 소진 시 충전 버튼을 누르고 앱 내 결제를 완료한다.
- **디자인 방향성:** 신뢰감을 주는 골드/블루 테마 결제 모달.
- **기술적 구현 방안:** PortOne(포트원) 연동, Database RPC (Ticket Credit).
- **데이터 로깅:** `pay_click`, `pay_success`, `pay_cancel`.
- **리스크:** 결제 연동 키 발급 지연(P0).

## 피쳐 (4): 결과확인 (Result)
- **배경:** 유저가 가장 기대하는 'Wow Moment'.
- **목표:** 실사화된 아기 모습에 대한 감동 극대화.
- **가설:** 정면/미소/수면 3종 세트 제공 시 SNS 공유 의사가 높아질 것이다.
- **핵심 유저 스토리:** 유저는 생성된 3장의 사진을 확인하고 고해상도로 저장한다.
- **디자인 방향성:** 명품 갤러리 스타일 레이아웃, Before/After 비교.
- **기술적 구현 방안:** SDXL 생성 이미지 S3/Storage 서빙, Realtime State.
- **데이터 로깅:** `result_view`, `image_download`, `share_click`.
- **리스크:** AI 생성 이미지 품질 편차.
