# [스케줄가재] 취합 보고: 현미경 설계 재검토 및 백로그 개편

## 1. DEV가재(개발)의 '현미경 설계' 현실성 재검토 (by 스케줄가재)
DEV가재 개발자가 수립한 `src/feature/{feature_name}/{domain|data|presentation}` 방식의 **Strict Clean Architecture**는 장기적인 유지보수에는 유리하나, **D-7 런칭**이라는 현재의 비즈니스 속도와는 정합성이 떨어집니다.

*   **진단 결과:** "과잉 설계(Over-Engineering)". 현재 백엔드(Supabase)와 프론트엔드(Next.js)가 직접 통신하는 구조에서 UseCase와 Repository 인터페이스를 일일이 분리하는 것은 개발 속도를 2배 이상 늦추는 병목입니다.
*   **조치 사항:** '현미경 설계'를 즉시 중단하고, **"Pragmatic Clean Architecture"**로 선회합니다.
    *   `src/feature/` 하위에 복잡한 레이어 분리 대신, `component`, `hook`, `service` 정도로 단순화하여 **기능 중심(Feature-driven)** 개발 속도를 확보하겠습니다.
    *   DEV가재 개발자에게는 아키텍처 토론보다 **'Hero Section 및 업로드 파이프라인'** 코드 구현에 집중할 것을 명령했습니다.

## 2. 개발 백로그 전면 수정 (with 기획가재 PO가재)
비즈니스 본질인 '일회성 경험'과 '압도적 바이럴'을 중심으로 백로그의 우선순위를 재정렬했습니다.

*   **삭제/후순위:** '리텐션' 관련 기능(회원 등급, 재방문 유도 푸시 등) 삭제. 결제 PG 연동(P1으로 하향).
*   **최우선(P0) 추가:** 
    *   **Viral Engine:** 인스타그램 스토리 최적화 '출생 예정 증명서' 카드 생성 기능.
    *   **Smoke Test:** 실제 결제 전 '구매 의사 확인' 버튼 및 데이터 로깅.
    *   **Concierge MVP:** PG 연동 전 무통장 입금 및 수동 알림톡 시스템.
*   **수정된 백로그 상세:** `docs/BACKLOG.md`에 반영 완료.
