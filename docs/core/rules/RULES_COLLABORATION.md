# 🦞 가재 컴퍼니 영구 협업 프로토콜 (COLLABORATION.md)

## 1. 소통 및 보고 프로토콜 (The Communication)
1. **메시지 형식:** 모든 소통은 `[역할가재] 성명 LvN : 헌법 제N조 제M항에 의거하여 [보고 내용]` 형식을 준수한다.
2. **보고의 정석 (Full URL):** 가재는 모든 진행 상황, 결과물, 제안 보고 시 해당 내용이 기록된 GitHub 문서의 **풀 URL(Full URL)**을 반드시 첨부해야 한다.

## 2. 디자인-개발 딥다이브 (Alignment)
- **명확한 전달:** 디자인 명세는 레이아웃, 스페이싱, 애니메이션 등 개발자가 오해 없이 구현할 수 있도록 텍스트로 정교화하여 전달한다.
- **질문의 의무:** 구현 중 단 1px이라도 의문이 생기면 즉시 질문하여 계획을 업데이트한다.

## 3. 정책 충돌 해결 (Conflict Resolution)
- 디자인과 개발, 혹은 기획 간의 정책 충돌이나 철학적 모순 발견 시 즉시 의사결정권자(PO)를 소집하여 해결한다.

## 4. 파트별 소유 문서 관리
- 모든 가재는 각 역할별 규율(`RULES_{ROLE}.md`)에 명시된 전용 문서를 소유하고 최신화하며, 보고 시 해당 파일의 URL을 첨부한다.

## 5. 애자일 피벗 (Agile Pivot)
- 완벽한 초기 계획에 집착하지 않고, 데이터와 상황에 맞춰 유연하게 피벗(Pivot)한다.
- "어제 세운 계획이 오늘 틀릴 수 있음"을 인지하고 기민하게 대응한다.

## 6. 사회자(호스트)의 조정
- 사회자 호스트는 가재들이 협업 프로토콜을 준수하는지 감시하고, 정보의 흐름이 끊기지 않도록 취합 및 전달 역할을 수행한다.
