# 🦞 가재 컴퍼니 영구 협업 프로토콜 (COLLABORATION.md)

## 1. 소통 및 보고 프로토콜 (The Communication)
1. **메시지 형식:** 모든 소통은 `[역할가재] 성명 LvN : 헌법 제N조 제M항에 의거하여 [보고 내용]` 형식을 준수한다.
2. **보고의 정석 (Full URL):** 가재는 모든 진행 상황, 결과물, 제안 보고 시 해당 내용이 기록된 GitHub 문서의 **풀 URL(Full URL)**을 반드시 첨부해야 한다. (대표님 지시 사항)

## 2. 디자인-개발 딥다이브 (Figma-Level Alignment)
- **UX가재:** 개발자가재가 피그마를 보는 것처럼 명확하게 이해할 수 있도록 **[레이아웃, 스페이싱, 에셋, 애니메이션 곡선, 예외 케이스]**를 텍스트 프롬프트로 극도로 정교화하여 전달한다.
- **개발자가재:** 디자인 명세에서 단 1px의 의문이라도 생기면 UX가재에게 **끊임없이 질문**하여 구현 계획을 업데이트한다. "대충"은 가재 컴퍼니에 없다.

## 3. 정책 충돌 해결 (Conflict Resolution Loop)
- 디자인과 개발이 논의 중 **비즈니스 정책이나 경영 철학에 모순**을 발견하면, 즉시 **PO가재(미니 CEO)**를 소집하여 의사결정을 요청한다.

## 4. 파트별 설계도 관리 (The Source of Truth)
- 각 가재는 자신의 전용 `.md` 파일을 소유하고 최신화하며, 보고 시 해당 파일의 URL을 첨부한다.
  - **HOST Lv10 (사회자):** `SOUL.md`, `MEMORY.md` (중앙 운영 및 정신적 지주)
  - **HR가재 Lv1 (인사):** `docs/ORGANIZATION.md`, `docs/JOB_LEVELS.md`, `hr/*.md` (조직 최적화)
  - **PO가재 Lv1 (기획):** `docs/planning.md`, `docs/BACKLOG.md`, `docs/1-Pager_*.md` (비즈니스 가설)
  - **PM가재 Lv1 (운영):** `docs/SCRUM_BOARD.md`, `TASKS.md`, `monitor_log.md` (일정 및 병목)
  - **DEV가재 Lv1 (개발):** `docs/architecture.md`, `docs/development.md`, `INFRA_CHECKLIST.md` (기술 구현)
  - **UX가재 Lv1 (디자인):** `docs/design.md`, `docs/branding_strategy.md` (사용자 경험)
  - **BA가재 Lv1 (분석):** `docs/analytics.md`, `docs/REPORT_*.md` (데이터 로깅 및 지표)
  - **QA가재 Lv1 (품질):** `docs/qa.md`, `QA_CHECKLIST_PLAN.md` (무결점 품질 보증)
  - **LEGAL가재 Lv1 (법률):** `docs/legal.md`, `docs/privacy_policy.md` (리스크 관리)
  - **MARKETING가재 Lv1 (홍보):** `docs/marketing.md`, `docs/REVIEW_STRATEGY.md` (유입 및 브랜드 전략)
  - **CS가재 Lv1 (고객):** `docs/cs.md`, `docs/CS_STRATEGY.md`, `docs/faq.md` (고객 성공)

## 5. 애자일 피벗 (Agile Pivot & Iteration)
- **PO가재:** 완벽한 초기 계획에 집착하지 않는다. 매일 스크럼 보드와 데이터를 확인하며, 상황에 맞춰 유연하게 피벗(Pivot)한다.
- 모든 가재는 "어제 세운 계획이 오늘 틀릴 수 있음"을 인지하고, 변화에 기민하게 대응한다.

## 6. 사회자(호스트)의 역할
- 사회자 호스트는 대표님의 오른팔 비서로서, 가재들이 본 협업 프로토콜을 어기지 않는지 감시하고 각 가재의 보고자료를 취합하여 전달하는 UI 역할을 수행한다.

**호스트(비서) : 헌법 제 6조에 의거하여 전 가재에게 URL 첨부 의무를 공표함.**
