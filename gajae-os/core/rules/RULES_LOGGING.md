# 🦞 가재 컴퍼니 대화 기록 헌법 (RULES_LOGGING.md)

본 문서는 가재 컴퍼니 내의 모든 의사소통 정보를 자산화하기 위한 기록 표준입니다.

## 1. 기록 저장 구조
- **경로:** `logs/dialogue/YYYY-MM-DD.md`
- **관리 방식:** 매일 새로운 파일을 생성하며, 대화 발생 시 실시간 또는 세션 종료 후 즉시 기록한다.

## 2. 기록 포맷 (Standard Format)
모든 로그는 아래 형식을 엄수하여 시간 순서대로 기록하며, **각 발언 이후에는 가독성을 위해 반드시 한 줄의 개행(Newline)을 둔다.**
- `[yyyy-MM-dd HH:mm:ss] [SessionID: xxxxx] 화자 : 메시지 내용`
- **SessionID:** 각 가재가 `spawn`될 때 부여받는 고유한 세션 UUID의 앞 8자리를 기입하여 지능의 독립성을 증명한다.
- **화자 구분:** 
  - 대표님: `낭만코딩`
  - 사회자: `HOST (Pi)`
  - 가재: `[직무] 이름` (예: `[PM] 최현우`)
- **세션 구분:** 미팅이나 특정 업무 세션의 경우 `--- [Session: 주제명] ---` 구분선을 사용한다.

## 3. 기록 대상
- **External:** 대표님과의 모든 대화(Telegram), 사회자(Host)의 모든 보고 메시지.
- **Internal:** 가재들 간의 업무 협의(Sessions) 및 가재들의 **내부 추론 과정(Internal Thoughts/Monologue)**.

## 4. 자율 커밋 및 무결성
- **커밋 주기:** 모든 가재는 보고서 작성 또는 대화 기록 추가 후 즉시 **자율 커밋(Self-Commit)**을 수행하여 기록의 손실을 방지한다.
- **삭제 금지:** 이미 기록된 로그는 어떠한 경우에도 삭제하거나 사후 가공할 수 없다. (History Integrity)

---
**지휘 지침:** "기록되지 않은 지능은 존재하지 않는 것과 같다." 모든 공정은 기록을 통해 증명한다.
