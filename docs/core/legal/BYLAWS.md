# 📜 가재 군단 하위 법령 (BYLAWS.md) v2.1

본 문서는 **통합 헌법 v26.0**을 보좌하며, 가재 군단 실무의 효율과 무결성을 극대화하기 위한 구체적 행동 지침을 정의한다.

---

## 제 1 조 (연대기 Append-Only 원칙)
1. 모든 대화 연대기(`docs/chronicle/daily/`)는 반드시 **'추가(Append)'** 방식으로만 작성한다. 
2. 이미 기록되어 커밋된 과거의 로그를 사후에 수정하거나 삭제하는 행위를 엄격히 금지한다.
3. 이는 지능의 진화 과정을 왜곡 없이 보존하기 위한 불변의 기록 원칙이다.

## 제 2 조 (지능형 자산 동기화 프로토콜: Intelligent Sync)
1. **Fresh Read before Write**: 가재는 파일을 쓰기(Write/Edit) 직전에 반드시 해당 파일을 다시 읽어(Read), 자신이 생각하는 원본 데이터가 최신 상태인지 확인해야 한다.
2. **Conflict Resolution**: 만약 연산 도중 타 가재에 의해 데이터가 변했을 경우, 자신의 결과물을 즉시 덮어씌우지 말고 최신본과 '지능형 병합(Merge)'을 수행해야 한다.
3. **Atomic Push**: 모든 깃 푸시 전 반드시 **`git pull --rebase`**를 수행하여 물리적 충돌을 예방한다.

## 제 3 조 (인덱스 지도 자동 생성)
1. 모든 하위 디렉토리의 `INDEX.md`는 수동 작성을 지향하고, 시스템 파싱 스크립트를 통한 자동 생성을 원칙으로 한다.
2. 인덱스 내의 모든 자산은 상대 경로를 통해 연결되어야 하며, 끊어진 링크 발견 시 즉시 인시던트를 발령한다.
