# 🦞 가재 컴퍼니 디자인 및 UX 헌법 (RULES_DESIGN.md)

이 규칙은 가재 컴퍼니의 모든 디자인 프로젝트에 예외 없이 적용되는 영구 규칙입니다.

## 1. 디자인 시스템 준수 (Design System First)
- 모든 UI는 프로젝트별로 정의된 **디자인 시스템**을 기반으로 제작한다.
- 텍스트, 폰트, 컬러, 위젯(버튼, 다이얼로그, 바텀시트 등) 컴포넌트를 사전 정의한다.
- **개발 통제:** 개발자가 임의로 처음 보는 스타일을 사용하지 않도록 강력하게 통제한다.

## 2. 시각적 방향성 가이드 (Visual Direction)
- **Vibe:** Clean, Professional, Safe, Emotional.
- **Typography:** 각 언어 및 도메인에 적합한 서체(Noto Serif, Pretendard 등)를 선정한다.
- **Color Strategy:** 기본 베이스 컬러와 강조(Accent) 컬러의 대비를 명확히 하여 신뢰감을 전달한다.

## 3. 명세 전달 (Figma-Level Documentation)
- 디자인 시스템을 베이스로 피그마 수준의 상세 `.md` 명세서를 만들어 개발자/기획자와 소통한다.
- 레이아웃, 스페이싱, 에셋, 애니메이션 곡선, 예외 케이스를 텍스트로 정교화한다.
- 모든 에셋은 웹 최적화 포맷(WebP 등)을 권장한다.

## 4. 디자인 QA (Visual Integrity)
- 개발자의 구현 결과물을 보고 실제 명세와 일치하는지 **디자인 QA**를 필수적으로 진행한다.
- 픽셀 단위의 오차, 애니메이션 타이밍, 상태별 UI 변화가 의도대로 구현되었는지 검증한다.
- 모바일 우선(Mobile-First) 대응 및 반응형 중단점을 확인한다.
